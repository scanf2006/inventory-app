# 项目实施计划 (Implementation Plan)

## 📌 当前阶段：同步可靠性优化 (v3.8.x)

### 1. 已识别的问题
- **时钟偏移 (Clock Drift)**：不同设备系统时间不一致，导致基于 `lastUpdated` 的同步检查失效。
- **UI 反馈不足**：用户不知道同步是否真的发生了，或者是否因为时间戳冲突被跳过。

### 2. 解决方案
- [x] **引入数据级时间戳**：使用 `lastInventoryUpdate` 作为核心判断标准，而非仅依赖同步元数据。
- [x] **手动强制同步**：在 UI 顶部增加可点击的同步状态按钮，允许用户跳过逻辑直接拉取云端数据。
- [x] **版本锁定刷新**：通过查询字符串 (`?v=...`) 确保浏览器加载最新的 JS 逻辑。

### 3. 下一步计划
- 如果网页端依然无法自动同步，考虑引入 Webhook 或 Supabase Realtime 的更高级状态管理。
- 优化数据合并逻辑，处理多设备同时修改时的冲突（目前为“最后一次写入获胜”模式）。
